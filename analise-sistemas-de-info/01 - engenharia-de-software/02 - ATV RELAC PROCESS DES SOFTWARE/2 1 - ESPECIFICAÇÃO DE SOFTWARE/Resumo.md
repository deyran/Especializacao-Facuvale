# Especificação de Software - Resumo

A especificação de Software é uma fase crítica que define o sucesso ou insucesso de um projeto. Ela envolve a documentação detalhada dos requisitos para que desenvolvedores e usuários tenham um entendimento claro do sistema a ser construído.

## Definições e Objetivos

- Compreender os serviços necessários ao sistemas
- Identificar restrições para o desenvolvimento
- Documentar os requisitos do sistema para conhecimento do escopo da solução

## Importância

- A Especificação de Software representa o sucesso ou fracasso do Processo de Software
- Requer responsabilidade e conhecimento para efetivação do processo

## Atividades do Processo

- *Estudo de Viabilidade* 

  * Investiga se a ferramenta proposta atende as necessidades do cliente

  * A investigação leva em consideração

    1. Ambiente computacional (hardware/software)
    2. Clientes
    3. Orçamento
    4. Ambiente de negócio

- *Modos de Levantamento*
  
  1. Entrevista
  2. Questionário
  3. Reuniões
  4. Observações
  5. Análise documental

- *Elicitação e Análise de Requisitos*

  Interagir com usuários para:
   
   1. Identificar necessidades
   2. Impulsionar o desenvolvimento
   3. Possibilide de usar prototipagem   

- *Especificação de Requisitos*

    Registrar as informaçõe levantadas, divididas em:

      1. Requisitos de Usuário: Abstrações das necessidades do usuário
      2. Requisitos de Sistema: Funcionalidades a serem implementadas

- *Validação de Requisitos*
  
    1. Constatar se os requisitos serão atendidos pelo sistema
    2. Uma vez constatado, permirtir correções na especificação
    3. Feedback do cliente confirmando o cumprimento dos requisitos
